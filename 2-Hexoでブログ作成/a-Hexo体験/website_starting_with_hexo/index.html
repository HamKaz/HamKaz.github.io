Hexoでサイト作成

Hexoとは？

静的サイトジェネレーターの一つで、JavaScriptで記述する。

静的ページジェネレーター - Hexo
言語 - Javascript
エンジン - Node.js
パッケージマネージャー - npm

ホスティングするサーバーでリアルタイム処理を要しないサイトを作成する場合に使用する。

アクセスの高速レスポンス、動的要素がないことにより脆弱性が最小化できるなどのメリットがある。

Node.jsとは？

Node.jsは、JavaScriptで書かれたプログラムの実行環境で、Googleにより開発されました。

サイトジェネレーターの仕組みを理解

静的サイトジェネレーターとは？

HTMLを変換・生成するツールを指し、生成したHTMLをホスティングサービスのサーバーにアップロードしておきます。従来のWordpressではブラウザからのリクエストがあってからHTMLを生成しますが、静的サイトジェネレーターではアップロード前にサイトページ全部をあらかじめ生成しておき、サーバーにデプロイする形を採ります。

WordpressではPHPで動的に生成ルールを記述しますが、静的サイトジェネレーターは、各種の言語でHTMLの生成ルールを決めておき、サイトの共通要素、各個別ページの要素ごとにHTMLを生成、CSS・JSとともに出力します。

静的サイトジェネレーターの理解

近年主流のプログラミング言語にチャレンジする上で、いろいろな静的サイトジェネレーターを試してみました。プログラミング目的に合わせて様々な言語によるジェネレーターが作成されています。

  • Hexo : Node.js : npm
  • Gatsby : React.js : yarn
  • Hugo : Go lang : linuxbrew
  • Jekyll : Ruby : gem
  • Pelican : Python : pyenv, pipenv

ブログサービスとの比較

ナレッジコミュニティの限界

noteやQittaなどのナレッジコミュニティでブログを始めると、運営側の都合で、IPなどの個人情報が公開されてしまったり、認証の不具合により利用できなくなったりします。

動的サイト生成との比較

プロユースで環境構築が大変。デフォルトのままでは動作が限定される。常日頃からのメンテナンスが必須。動作が重たい。セキュリティリスクが大きい。

  • Wordpressは環境依存性が高すぎ、負荷も大きすぎ、サーバー代が高くつく
  • 原稿の管理に負担が大きく、ファイル転送に手間がかかる
  • サーバーアプリやデータベースの管理（保守・メンテ）に手がかかりすぎ、セキュリティの維持も大変である

ブログをカンタンに構築し、負担なく運用するには、静的サイトジェネレーターとホスティングサービスの組み合わせがオススメ。

「技術メモを残す」「自前で小さく運用する」、「静的サイトジェネレーターでHTMLを生成」「リモートでHTMLをホスティング」、日常のコスト（管理手間）が小さく、ランニングコスト（サーバー負荷）が比較的小さくすむ。

ホスティングとデプロイ

ホスティング

あらかじめ静的サイトジェネレーターで生成したHTMLは、サーバーで公開し運用することになります。静的サイトジェネレーターで生成されたHTMLをホスティングするためには、最低限のWebサーバー機能で十分になります。

無料でホスティングできるサービスを調べてみたところ、GitHub Pages、NetLify、Heroku、GitLab、Firebaseなどがあることが分かりました。

HexoブログをGitHub Pagesで最速公開する

https://tech.qookie.jp/posts/hexo-deploy-github-pages-fast/

デプロイ

ジェネレートで生成されたHTML、CSS、JSを、ホスティングサービス上に直接デプロイし公開・運用することにしました。

GitHub Pagesを含むホスティングサービスでは、アフィリエイトの運用が許可されています。

実際多くの人が実際に運用されているようで、技術ブログを長く続けるモチベーションが得られることも考慮に入れました。

まとめ

今回はかんたんなブログ作成と運用を、静的サイトジェネレーターとホスティングサービスで手軽に実現できることについてまとめてみました。

静的サイトジェネレーターでは、ローカルPCでブログ生成（ジェネレート）、GitHubでブログデータ保守（バックアップ）、ホスティングサービスGitHub Pagesでブログ公開（デプロイ）することができます。

Hexoは、Javascriptを学習する良いきっかけとなりました。

今後は、書き溜めた技術メモを、ブログ原稿としてリファインし、公開していきたいと思います。

Hexoテーマの書かれたJavascriptを書き換えることで、生成されるHTMLの見た目や動作を自分の自由な形に変更できます。

  • Web上に文献がたくさん存在し先人の知識や経験を参照できるとともに、身近に尋ねる人がおらず、Web上の文献だけで問題解決できる
  • Linuxでの構築コスト（手間と時間）が最も少なくて済み、パッケージ管理方法がLinuxに向いたものであること
  • Javascript、Node.jsをコスト（導入・学習・金銭）をかけずに学習でき、Linux上で不具合なく動作させることができること
