npm - Node.jsのパッケージ管理

npmには２つの意味があり、一つはオープンソースパッケージが管理されているオンラインリポジトリを指し、もう一つはパッケージを管理するためのコマンドラインツールを指します。

What is npm? （Node.js公式ページ）

Node.jsのパッケージ管理 npm

Node.jsのパッケージはコマンドラインツールnpmを使ってインストールします。npm自体はNode.jsをインストールすると同時にインストールされます。

npm installコマンドでインストールされたパッケージは管理ファイルpackage.jsonで管理され、コマンド実行時のディレクトリ（プロジェクトのルートディレクトリ）に配置されます。

もう一つのパッケージ管理 Yarn

Node.jsが普及するにつれて、npmにはインストール速度やセキュリティの問題があることが分かりました。そこでFacebook、GoogleなどによってYarnというパッケージ管理方法が新たに発表されました。

Yarn：Facebook発のパッケージマネジャーはnpmに代わるスタンダードになるか

Yarnはnpmと互換性のあるパッケージ管理方法で管理ファイルpackage.jsonに記録するとともに、yarn.lockファイルを生成してモジュールの管理を行います。

Yarn自体はnpmコマンドによりnpm install -g yarnでインストールします。

コマンドラインnpm実行時の警告・エラー

1. “found * low severity vulnerability”

インストールしたパッケージモジュールに脆弱性があると公表されている場合、このようなメッセージが表示されます。npm auditコマンドで、解決方法が存在するか確認できます。

a) 解決方法が存在する場合

npm auditコマンドを実行すると、解決方法の有無がメッセージで表示されます。表示内容に従ってコマンド実行すると、脆弱性のあるモジュールが更新されます。

未知の脆弱性が存在する場合には、npm auditで解決できるとは限らない場合があります。

b) 解決方法が存在しない場合

npm audit fixを実行して、より脆弱性の低い状態に更新しておきます。低脆弱性でありLAN内で使用する場合にはそのままにしておきます。

上記の例では、呼び出し元のoptimistが、minimistの古いバージョンに依存しているといった脆弱性警告メッセージが表示されています。

2. SKIPPING OPTIONAL DEPENDENCY: fsevents

MacOSに依存性のあるパッケージモジュールをインストールした場合、次のようなメッセージが表示されます。

対策としては、OPTIONAL DEPENDENCYと表示されたオプションモジュールを除外するため、npm install -fコマンドで除外インストールします。

3. fundの表示の意味は？

パッケージの開発者への寄付を募るメッセージであり、npm fundコマンドを実行すると、どのパッケージが寄付対象であるか示されます。

まとめ

npmとyarnはどちらか一方を使用しなければならないというわけではなく、混在して使用できます。

Yarnを用いると、パッケージモジュールのインストールが早くなり、インストール時のスクリプト実行が抑えられてセキュリティが確保されることから、yarnを使う動きが広がっているようです。
