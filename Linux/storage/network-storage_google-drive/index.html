Google Drive - オンラインストレージ

Google Driveはブラウザベースのオンラインストレージで、とても使い勝手が良くできています。

Googleドライブのファイルの移動方法は、ファイルをブラウザにドラッグアンドドロップしてアップロード・ダウンロードします。

リモート・ローカル間をアップロード・ダウンロードのワンクッションを経ずに移動させたくなり、オンラインストレージをローカルにマウントする方法を調べてみました。

オンラインストレージのマウントとは？

調べてみたところ、Google DriveはLinux上の仮想ファイルシステムとしてマウントして使うことができます。

参考サイト：ubuntuで利用できるオンラインストレージまとめ

この記事を参考に調べたところ、Debian10にもgnome-online-accountsを使ってGoogle Driveへ接続できることが分かりました。

Debian10でオンラインストレージへマウント接続するには、gnome-online-accountを使用します。

インストールに必要なコマンド

apt install

インストール手順

使用方法

Googleアカウントの登録

GUIからの起動は、設定 > オンラインアカウントから行います。

Gnome Online Accountsでは、いくつかのオンラインストレージのアカウントを登録でき、その中にGoogleアカウントも含まれています。

使用方法

OSに用意されているアイコン「ファイル（Files）」をクリックして開くと、左側のツリーの中に自分のアカウント名が表示され、Google Driveへマウントされていることが確認できます。

まとめ

Googleドライブをローカルディスクのように扱うことは、アップロード・ダウンロードを意識しなくても良くなり、便利にファイル移動できるようになりました。

しかしネットワーク負荷の状況、Google Driveのサーバーレスポンスのタイミングにより、Google Driveヘの書き込みに大幅な遅延が発生することがあります。

またGoogle Driveへの接続には、常時ネットワーク接続を要求されるため、オフラインでの使用に課題が残りました。
