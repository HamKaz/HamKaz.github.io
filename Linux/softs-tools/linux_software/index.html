Linuxをベースとしたプログラミング開発機

Linuxマシン上にネットワークアプリケーションのプログラミング開発環境を構築する目的で、いろいろなソフトウエアとツールをインストールしました。

Linuxでは、全てのソフトウエアはリポジトリでインストール履歴・依存関係をバージョン管理します。バージョン管理方法とコマンド操作はソフトウエアの配布元により異なります。

今回、Linux Debian10をベースとしてソフトウエアとツールをインストールするために必要な知識と環境を記録しました。

リポジトリとバージョン管理

Linuxのソフトウエアやツールはリポジトリによりバージョン管理し、リポジトリにはインストールに必要なソフトウエア情報と依存関係を記録します。

OSのディストリビューション、例えばDebianではaptによりOS機能と付属ソフトウエアの配布を行いますが、Debianの管理範囲外のソフトウエアをインストールする場合、セキュリティの観点から次の方法でインストールします。

  1. ソフトウエア配布元から認証キーを取得
  2. リポジトリを更新
  3. リポジトリ情報に従いソフトウエアをインストール

インストールに必要なコマンド

いろいろな会社や団体でソフトウエアのインストール方法・管理方法が異なるため、整理する意味で、「ソフトウエア - バージョン管理 - 使用するコマンド」の順に列挙しました。

ブラウザ - Chrome

debインストール : wget, apt install

aptインストール : wget, apt-key, sh, apt-get update, apt-get install

JavaScript環境 - Node.js

aptインストール : curl, apt-get install, apt-key, tee, apt-get update

エディタ - Atom Editor, Visual Studio Code, Typora

Atom Editor

aptインストール : wget, apt-key, sh, apt-get update, apt-get install

Visual Studio Code

aptインストール : curl, install, sh, apt-get update, apt-get install, apt-transport-https

Typora

aptインストール : wget, apt-key, add-apt-repository, apt-get update, apt-get install

コマンド種別の整理

ソフトウエアのインストールに必要なコマンドを、作業別に仕分けしました。

リポジトリ管理・インストール

apt、apt-get update、apt-get install、apt-key、add-apt-repository、apt-transport-https

ダウンロード

wget、curl

入出力処理

install, sh, tee, echo

まとめ

WindowsやMacのインストールでは、インストールパッケージをクリックして行い、また最新バージョンへのアップデートもWindows UpdateやApple Storeで自動的に行われます。

特にWindowsはその万能性が故に様々な問題を引き起こすことがあります。
・不適切なアップデートにより動作不安定を引き起こす
・バックグラウンド処理により動作が重くなる
・バージョンを固定してインストールしたくない
・レジストリの正しい書き換えがなされない　など

Linuxは使用者の意図に従い、適切な動作・マシン性能に見合った安定動作・クリーンなバージョン管理ができますが、一方で、リポジトリ管理が手動となり、管理手間が増大するという点があげられます。

OS選びはとても重要です。目的に応じた市場・分野・特性に合わせてOSを選定し、プログラミング環境の構築を行う必要があります。
