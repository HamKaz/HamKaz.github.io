開発環境の構築は難しい

アプリケーションソフトウエアのコーディングは、開発からデプロイまで一貫した環境下で行う必要があります。

開発者によって開発環境が異なったり、開発が進むにつれてプラットフォームの変更が生じた場合、アプリケーションの動作が不安定になったり、クラッシュ時にはリカバリ不可能になることも考えられます。

開発環境をできるだけハードウエアやOSに依存させないようにするためには、グループ内でバージョン管理し、依存性を同一にすることが求められます。

そこで仮想環境に開発環境のイメージを作成して、開発からデプロイまで仮想環境上で一貫して構築する方法が取られるようになってきました。

Linuxのコンテナ化技術で仮想環境を実現するDockerについて調べてみました。

仮想環境とは？

過去から使用されている仮想環境技術として、仮想マシンという考え方があります。

VMwareなどの仮想マシンでは、ホストOS上にソフトウエアで構築した仮想マシンを作成し、ゲストOSを仮想マシンにインストールして仮想環境を作り上げる手法です。

仮想マシンではGUIやハードウエア資産を制御できます。

一方Dockerは、Linuxのコンテナ技術を利用した仮想化技術で、アプリケーションを直接ホストOSで仮想実行する技術で、仮想実行環境をイメージ化することで、再配布や実行環境への移行が容易に出来るようになりました。

参考サイト：Docker入門（第１回）

Dockerでは、アプリケーションをOSから分離して構築することができ、Dockerの仮想化イメージを利用して、いろいろな環境が構築できます。
