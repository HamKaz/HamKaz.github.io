Linuxで環境構築

2020-05-13

  • home LinuxとNodejsでプログラミング Linux体験

Linux環境の構築は楽ではない

表題の通り、Linuxでの環境構築は楽ではない。なぜならLinuxの世界では正答は一つではないから。
答えを見つけやすくするため、LinuxのディストリからDebian系のUbuntuを選んだ。

統一感にこだわるなら

MacOSやWindowsにはかなわない、やはり大企業の開発したOSを選んだほうが無難だろう。
UNIXのオリジナルは大学や企業の一部の人間でないと使えなかったし、su権限の必要な操作は許可されないし。

ただしLinuxは万能ではない。見た目とウィンドウの使い方、フォントの美しさとバリエーション、表示フレームや操作感など劣る部分もたくさんある。そこを全てLinuxに求めるのは無理がある。

プログラミングのためのOS

ではLinuxはどんなことに有利なのか、最初はUNIXのコピーという成り立ちだったこともあり、ネットワーク向けのプログラミングだと私は考える。

一つはネットワークアプリケーション、もう一つはクロスプラットフォームアプリであろう。これらはLinux環境が向いている。
私は、ネットワークアプリケーションの取っ掛かりとしてNode.jsを選んだ。

パッケージとリポジトリ

プログラミングの世界、Linuxのソフトウエアはパッケージ管理、これを分かるようになることを、このブログでは強く言いたい。

プログラムの集合体がパッケージで、そのパッケージがリポジトリというところに置かれて管理される。

プログラミングの要（カナメ）は言語というよりも構成と管理なのだと。
みんな言語に興味は持つのだが、構成と管理に重きを置く解説が少ないのが残念。

とは言っても、リポジトリによるソフトウエアのバージョン管理、インストールに必要なソフトウエア情報と依存関係の把握、Linux系のコマンド操作、そのくらい覚えればプログラミングは始められる。

OSのディストリビューションによって管理方法は異なる。例えばDebian・Ubuntu系ではaptといったように。

自学自習は苦しい、必ず得るものがある

しかしLinuxは自己責任、まさに言葉の通りで難しい。当然管理は自分で行う。だから大変、だから楽しい。

Linuxは、自分で環境整備できるようになる、ネットワークプログラミングが分かるようになる、この２点がポイントである。

MacやWindowsとは違うから

正直、プログラミングで生み出す経済的効果に対し、MacやWindowsPCはハードウエアコストが過剰であると思う。金が儲かるようになって償却資産が必要になってからスペックアップすれば良い。

それまではLinuxをインストールした安い中古PCで、Web上にある参考資料を片手に自学自習すればいい。とても参考になる良いWebサイトがたくさんある。

ましてやマニュアル本が自分の環境に適合しているわけでもないし、最新版というわけでもない。ソフトのバージョンアップに本の内容がついてこないし。

工場での制御系PC？オフィスアプリ開発？なら、Visual Studio開発とSwiftなんかをバージョン固定して開発することが必要だが、それも本業・専業でなければ意味がないし、制限の多いMacやWindowsはもったいない。

いよいよLinux、さあトライしてみよう。Linuxができると知見が広がるから。
